## 1. [roadmap_zh.md](./roadmap_zh.md) 系列拆分

### 1.1 七篇别乱看：先把 Informed ML 分成两条主线
- 内容核心点：
  - 当前这套材料不是一条单线，而是两条互补主线。
  - 一条主线在问“规则怎样约束训练与推理”。
  - 另一条主线在问“知识本身有模糊性时，模型怎样更可信地表达结果”。
  - 不先分线，后面很容易把逻辑约束、粒化输出、知识正则混成一个概念。
- 建议讲解角度：问题导向

### 1.2 为什么 Survey 必须是第一站
- 内容核心点：
  - Survey 的作用不是给答案，而是给坐标系。
  - 它先帮你统一三个问题：知识从哪里来、怎么表示、注入到哪里。
  - 没有这张总地图，后面的 Logic-Net、Semantic Loss、DL2 很容易被误读成零散技巧。
  - 它是“先看世界地图，再看单篇论文”的那一步。
- 建议讲解角度：费曼

### 1.3 Logic 主线怎么推进：从 teacher 到 semantic loss 再到 declarative constraints
- 内容核心点：
  - Logic-Net 代表的是 `rule -> teacher -> student`。
  - Semantic Loss 代表的是 `rule -> exact semantic loss`。
  - DL2 再往前一步，把约束变成统一声明式接口，不只训练时能用，查询时也能用。
  - 这条线的本质是：同样是逻辑知识，进入优化器的接口在不断变化。
- 建议讲解角度：类比

### 1.4 Granular 主线怎么推进：从“满足规则”走到“结果是否可信”
- 内容核心点：
  - 这条线不是在问规则怎么写，而是在问模型为什么不能只给一个过于确定的点值。
  - Granular ML 先提出输出应该带范围、带可信度。
  - Fuzzy to Granular 继续把规则模型升级成 interval / fuzzy / probabilistic 输出。
  - Knowledge Landmarks 再把局部数据和全局知识放进统一训练框架。
- 建议讲解角度：问题导向

### 1.5 跨论文别只记公式：先问“知识是什么、注入哪里、优化谁”
- 内容核心点：
  - 真正有研究价值的阅读，不是背公式，而是先认清对象。
  - 第一问是：论文把知识当成逻辑、后验偏置、可行域，还是粒化对象。
  - 第二问是：知识进入 posterior、loss、feasible set、output representation，还是 regularizer。
  - 第三问是：模型真正优化的是参数、教师分布、反例变量，还是粒的边界。
- 建议讲解角度：费曼

### 1.6 七篇最后会汇到哪里：不是“加知识”，而是重新设计学习系统
- 内容核心点：
  - 前半条线更关注逻辑有效性，后半条线更关注可信表达。
  - 真正成熟的问题不是只会写约束，而是能不能同时处理“知识有效性”和“输出可信性”。
  - 这意味着任务损失、逻辑约束项、粒化正则项可能需要一起设计。
  - 未来真正有创造性的工作，常出现在这两个世界的交叉带上。
- 建议讲解角度：费曼

### 1.7 时间不够时，最短主线应该怎么抓
- 内容核心点：
  - 如果只想先抓一条最稳的主线，可以先读 `survey -> Logic-Net -> Semantic Loss -> Knowledge Landmarks`。
  - 前三篇帮你建立 informed ML 的经典逻辑注入链。
  - 最后一篇把视角拉向“局部数据 + 全局知识 + 可信表达”的研究问题。
  - 这条短线比“把七篇平均看一遍”更容易形成自己的判断。
- 建议讲解角度：问题导向

## 2. [survey_notes_zh.md](./survey_notes_zh.md) 系列拆分

### 2.1 不是“我有经验”就叫 Informed ML
- 内容核心点：
  - Informed ML 不是泛泛地说“我有先验”。
  - 真正算数的 prior knowledge 必须来自训练数据之外。
  - 这种知识还必须能被形式化，并且被显式接入学习流程。
  - 所以 informed ML 的门槛不是主观经验，而是外部知识能不能变成可操作对象。
- 建议讲解角度：问题导向

### 2.2 看懂 Informed ML，先抓住一张图：Data 之外还有 Prior Knowledge
- 内容核心点：
  - 普通机器学习主要依赖数据流。
  - Informed ML 多了第二输入源：prior knowledge。
  - 这份知识不是停留在研究者脑中，而是要进入 pipeline。
  - 所以它和普通 ML 的区别，不在模型名字，而在信息流结构。
- 建议讲解角度：费曼

### 2.3 先验知识要想真的“进模型”，至少得满足哪 3 个条件
- 内容核心点：
  - 第一，知识来源独立，不是从当前数据里又拟合出来的。
  - 第二，知识能形式化成规则、方程、图结构、仿真结果等对象。
  - 第三，知识要显式集成进系统，而不是只做模糊灵感。
  - 这三条条件，帮你快速筛掉很多“看上去像 informed ML、其实不是”的工作。
- 建议讲解角度：问题导向

### 2.4 Table 4 最该怎么读：不是分类表，而是一条条知识注入路径
- 内容核心点：
  - 这张表的核心句式是 `Source -> Representation -> Integration`。
  - 重点不是背方法名，而是看知识从哪里来、被写成什么、从哪个接口进系统。
  - 每一行还连着动机、中心思路、挑战和未来方向。
  - 一旦学会这样读，taxonomy 就从“表格记忆题”变成“理解工具”。
- 建议讲解角度：费曼

### 2.5 为什么 representation 才是 informed ML 最关键的接口
- 内容核心点：
  - 知识来源更像应用背景，注入位置更像工程接口。
  - 真正起桥梁作用的是 representation，因为它决定知识能否被模型吸收。
  - 同样是“知识”，写成规则、图、方程或仿真样本，后面的学习接口会完全不同。
  - 所以 informed ML 不是“有没有知识”，而是“知识被写成什么样”。
- 建议讲解角度：费曼

### 2.6 知识不是只能进 loss：很多人一开始就把 informed ML 想窄了
- 内容核心点：
  - 知识可以进入 training data，用来造样本或弱监督。
  - 知识也可以进入 hypothesis set，直接塑造模型空间。
  - 还可以进入 learning algorithm，变成 loss、正则或约束。
  - 甚至可以进入 final hypothesis，做后验修正和结果校验。
- 建议讲解角度：类比

### 2.7 Logic Rules 这一路，到底是在改结构、改概率，还是改训练
- 内容核心点：
  - Logic rules 不是单一方法族，而是至少有三层接口。
  - 一层是 KBANN 这类规则塑造结构的路线。
  - 一层是 MLN / PSL 这类规则定义概率依赖模板的路线。
  - 一层是 Semantic Loss、teacher-student 这类规则进入训练目标的路线。
- 建议讲解角度：问题导向

### 2.8 Knowledge Graph 不是“换个壳子的规则”
- 内容核心点：
  - Logic rules 更关心命题约束和 if-then 结构。
  - Knowledge graphs 更关心实体、关系和连接模式。
  - 图谱的价值常常不在“让某条规则成立”，而在“把关系结构作为第二信息源”。
  - 所以 graph 路线更容易走向 GNN、graph reasoning、distant supervision 和 graph validation。
- 建议讲解角度：类比

### 2.9 规则和图谱到底怎么选：这篇 survey 最实用的价值在这里
- 内容核心点：
  - 如果你的知识天然长成“如果……则……”或“不能同时成立”，规则更自然。
  - 如果你的知识天然长成“谁和谁有关、要不要用邻居和路径”，图谱更自然。
  - rules 更常约束模型，graphs 更常重构模型看世界的方式。
  - 学会这个判断后，survey 才真正从综述变成选题地图。
- 建议讲解角度：问题导向

## 3. [logic_net_notes_zh.md](./logic_net_notes_zh.md) 系列拆分

### 3.1 规则进神经网络，为什么不是“加个 rule loss”就完了
- 内容核心点：
  - 很多人一说知识注入，第一反应就是“再加一个约束损失”。
  - 但早期 neural-symbolic 路线还有“规则直接造结构”和“规则只做后处理”两种典型接口。
  - Hu 2016 真正想解决的，是在不改坏通用网络主体的前提下，让规则进入训练核心。
  - 它代表的是第三条路，不等于普通 penalty learning。
- 建议讲解角度：问题导向

### 3.2 Hu 2016 到底填了哪个坑
- 内容核心点：
  - 直接按规则造网络，表达力强，但结构太受规则形式约束。
  - 只把规则做成额外特征或后处理，规则又进不了优化核心。
  - Hu 2016 的答案是：先改当前预测分布，再把这个偏置蒸馏回参数。
  - 这让规则既参与训练，又不强迫你重写整个架构。
- 建议讲解角度：费曼

### 3.3 Logic-Net 先改的不是参数，而是 posterior
- 内容核心点：
  - 这类方法的主接口不是 loss，而是 posterior regularization。
  - 它先问“哪些候选输出应该被重新加权”，而不是“参数该罚多少”。
  - 规则违反多的状态在教师分布里被压低，守规则的状态被相对抬高。
  - 所以这一步改的是 belief geometry，不是直接改参数。
- 建议讲解角度：费曼

### 3.4 闭式教师分布 q* 到底是什么：它不是另一个模型，而是“被规则修正后的当前分布”
- 内容核心点：
  - 教师分布来自对当前学生分布的投影，而不是独立训练出来的第二网络。
  - 它同时要兼顾两件事：别离学生太远、又要更守规则。
  - 从形式上看，它是对当前输出世界进行指数重加权。
  - 用最小例子去看，本质就是把违例世界从高概率集合里挤出去。
- 建议讲解角度：类比

### 3.5 为什么还要 teacher-student 蒸馏这一层
- 内容核心点：
  - 如果只停在 posterior 修正，规则只是在当前时刻做分布整形。
  - 蒸馏的作用，是把这种整形后的偏置逐轮写回学生参数。
  - 学生最终拟合的不是硬标签，也不是纯教师，而是标签和教师的加权混合目标。
  - 这就是它和“测试时后处理”最本质的区别。
- 建议讲解角度：费曼

### 3.6 同一条规则，进 posterior、进 loss、进 feasible set，完全不是一回事
- 内容核心点：
  - 在 Hu 2016 里，规则先重写的是世界分布。
  - 在 Semantic Loss 里，规则直接变成“合法世界总概率”的损失项。
  - 在 DL2 里，规则还可能进一步变成可搜索的可行域。
  - 同一条逻辑约束没变，变的是它进入优化器的接口。
- 建议讲解角度：问题导向

### 3.7 Logic-Net 真正在优化什么：它更像投影-蒸馏交替，不像一个静态 loss
- 内容核心点：
  - 每一轮先生成教师，再固定教师更新学生。
  - 所以它不是对一个端到端静态目标直接做标准 SGD。
  - 更准确地说，它像一个交替优化或不动点迭代过程。
  - 真正该担心的问题，不只是“loss 怎么写”，而是“这个外层算子稳不稳定”。
- 建议讲解角度：费曼

### 3.8 Logic-Net 最容易出问题的 5 个地方
- 内容核心点：
  - 第一类问题是外层优化不稳定，teacher 会随 student 一起漂。
  - 第二类问题是规则权重敏感，因为它通过指数重加权改变 posterior geometry。
  - 第三类问题是规模一大就进入结构化推断难题。
  - 第四类问题是并非所有逻辑都能稳定 ground、连续化并推断。
  - 第五类问题是 teacher-student gap，teacher 的结构不一定能被 student 模型类完全学回去。
- 建议讲解角度：问题导向

### 3.9 如果继续做研究，Logic-Net 最值得改哪几个接口
- 内容核心点：
  - 可以改 projector，让规则投影更稳定，而不是只靠 KL 形式。
  - 可以把全局统一规则改成局部可信度不同的 granular 规则对象。
  - 可以把规则从输出端下沉到 feature level，缩小 teacher-student gap。
  - 也可以把昂贵 projector 摊销成可学习模块，降低复杂规则下的成本。
- 建议讲解角度：问题导向

## 4. [semantic_loss_notes_zh.md](./semantic_loss_notes_zh.md) 系列拆分

### 4.1 为什么普通交叉熵，管不住“有结构的输出”
- 内容核心点：
  - 普通交叉熵只回答“像不像标签”，不直接回答“整体输出来得合不合法”。
  - 一旦任务带有 one-hot、路径、排序这类结构，局部预测对了，整体仍可能违约。
  - Semantic Loss 的切入点，是把“输出必须满足某种结构”直接变成训练信号。
  - 所以它关心的不只是标签拟合，还关心可行输出空间。
- 建议讲解角度：问题导向

### 4.2 softmax 不是 exactly-one，这一步很多人一开始就搞混了
- 内容核心点：
  - softmax 描述的是“一个类别变量属于哪一类”的分布。
  - Semantic Loss 的经典设定常是 independent Bernoulli，每一位单独给出为真的概率。
  - `exactly-one` 的作用，不是重复 softmax，而是把概率质量从非法组合推回 one-hot 合法世界。
  - 不分清这两层对象，后面几乎所有公式都会读歪。
- 建议讲解角度：类比

### 4.3 Semantic Loss 一句话讲透：它优化的是“合法世界总概率”
- 内容核心点：
  - 先把逻辑约束看成一组合法输出，而不是一串公式文本。
  - 模型给出分布后，统计这个分布落在所有合法输出上的总概率质量。
  - 再把这部分质量取负对数，就得到可训练的损失。
  - 所以它不是盯某一条规则的局部真假，而是在问“合法世界总共占了多少概率”。
- 建议讲解角度：费曼

### 4.4 为什么它叫 Semantic Loss：关键不在公式写法，而在逻辑 meaning
- 内容核心点：
  - 只要两个公式逻辑等价，它们的 Semantic Loss 就应该一样。
  - 这说明它关心的是语义，而不是 CNF、DNF 之类的语法外形。
  - 当约束退化成单个变量真假时，它又会退化成普通 Bernoulli 交叉熵。
  - 所以它不是与传统监督割裂的新对象，而是把监督 loss 推到了结构层面。
- 建议讲解角度：费曼

### 4.5 用 4 分类例子看懂：一个 loss 怎么把概率往 one-hot 上推
- 内容核心点：
  - 4 类 `exactly-one` 的合法世界其实只有 4 个 one-hot 赋值。
  - `(1100)`、`(0000)` 这类状态在逻辑上都属于非法世界。
  - Semantic Loss 惩罚的不是“某一维太高”，而是“模型把太多质量放在了非法世界”。
  - 一旦合法总质量上升，loss 就会下降，这就是它的训练信号。
- 建议讲解角度：类比

### 4.6 无标签样本也能训练？Semantic Loss 最反直觉的地方在这
- 内容核心点：
  - 无标签样本虽然不知道具体类别，但仍然知道输出不该长得乱。
  - 如果一个样本同时高亮多个类别，Semantic Loss 依然会给出惩罚和梯度。
  - 它给的是结构监督，不是类别监督。
  - 这就是它在半监督设置里有价值的根本原因。
- 建议讲解角度：问题导向

### 4.7 Semantic Loss 罚的不是局部规则违背度，而是在重排整个分布
- 内容核心点：
  - 很多规则方法关注“当前点违反规则有多严重”。
  - Semantic Loss 关注的是“整个分布在合法世界集合上放了多少质量”。
  - 所以它是分布级、全局耦合的，而不是点态、局部的。
  - 这也是它和很多 soft logic surrogate 最大的分野。
- 建议讲解角度：费曼

### 4.8 Semantic Loss、Logic-Net、DL2，为什么不是一回事
- 内容核心点：
  - Logic-Net 是规则先进入 posterior，再蒸馏回参数。
  - Semantic Loss 是规则直接进入损失，优化合法世界总概率。
  - DL2 更常优化连续 surrogate 的违反幅度，或者直接限制可行域。
  - 三者都在用逻辑，但它们真正改写的是优化器的不同接口。
- 建议讲解角度：问题导向

### 4.9 Semantic Loss 不是银弹：它最强的地方，也正是它最贵的地方
- 内容核心点：
  - 它的优势是语义精确，真的在优化满足约束的总概率质量。
  - 代价是复杂约束下，合法世界会爆炸式增长，计算可能要靠逻辑编译或 WMC。
  - 它更适合命题级、离散、结构清晰的输出。
  - 如果约束本身写错，模型也会被非常坚定地带偏。
- 建议讲解角度：类比

## 5. 发布策略建议

### 5.1 建议结论：不要等七篇全读完再发，也不要“看一点就发一点”
- 内容核心点：
  - 更合理的做法是按“局部闭环”边学边发。
  - 也就是先把一个小闭环真正吃透，再开始发对应系列。
  - 目前最适合先形成闭环的是 `survey -> Logic-Net -> Semantic Loss`。
  - 这样既保留学习效率，也能避免内容显得零散和不稳。

### 5.2 为什么不建议等七篇都学透再发
- 内容核心点：
  - 从学习角度看，讲给别人听本身就是最强的理解检验。
  - 从节奏角度看，等全学完再发，很容易一直拖到没有输出。
  - 从内容角度看，小红书更适合连续更新，不适合长期憋一个“大而全”的总答案。
  - 你真正需要的不是“全部学完”，而是“每次发出来的那个局部已经站得住”。

### 5.3 为什么也不建议完全边看边发
- 内容核心点：
  - 人设风险不是“边学边发”本身，而是“边学边下结论”。
  - 如果还没搞清和上一篇的关系、方法边界、局限性，就容易显得不稳。
  - 这种不稳不会增加真实感，反而会削弱可信度。
  - 所以关键不是发得早或晚，而是有没有设最小发布门槛。

### 5.4 我更建议你采用的最小发布门槛
- 内容核心点：
  - 能用自己的话说清这篇论文解决什么问题。
  - 能说清它和上一篇相比，接口或优化对象到底变了什么。
  - 能指出至少两个局限或最容易误读的点。
  - 能拿出一个最小例子、图示或 running example，把机制讲到别人能复述。
  - 做不到这四条时，先写内部笔记，不急着外发。

### 5.5 从人设角度，最稳的定位方式是什么
- 内容核心点：
  - 不要把自己包装成“给最终定论的人”，而要明确定位成“正在系统拆论文的人”。
  - 你的优势不是权威口吻，而是结构化、持续、能把难论文讲清。
  - 用“这是我当前最稳定的理解”比“这篇论文就是这样”更稳。
  - 只要边界感清楚，边学边发不会伤人设，反而会形成可信的成长轨迹。

### 5.6 一个更现实的执行建议
- 内容核心点：
  - 第一阶段先发 `roadmap + survey`，把整个系列的人设和世界观搭起来。
  - 第二阶段连发 `Logic-Net` 和 `Semantic Loss`，形成一条清晰的逻辑主线。
  - 第三阶段等 `DL2` 和 granular 主线更稳后，再扩到第二条主线。
  - 每发一篇前，先在这份文档里完成拆分，再去写正式文案。

### 5.7 更适合你当前阶段的表达口吻
- 内容核心点：
  - 你的定位更适合是“认真搭学习地图的人”，不是“已经给领域下结论的人”。
  - 少用“本质就是”“根本不是”“一定是”这类封口式表达。
  - 多用“我目前的理解”“为了不看乱，我先这样分”“这是我的入门整理”这类有边界感的说法。
  - 可以明确告诉读者：这不是定论，而是当前学习阶段最稳定的一版理解。
  - 这样既不会显得虚，也不会给人“只看几篇就开始定性整个方向”的观感。
