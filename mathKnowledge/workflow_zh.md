# mathKnowledge 内容工作流（中文）

## 1. 内容放哪里

- `content/`：放源稿，适合长期修改、扩写、重组
- `docs/`：放将来站点直接发布的页面
- `templates/`：放统一写作模板

## 2. 建议写作顺序

1. 先在 `content/` 里写概念草稿
2. 整理成人话解释、最小例子、易错点
3. 再决定是否同步到 `docs/` 形成公开页面

## 3. 每篇最小结构

1. 这部分在研究什么
2. 核心定义的人话解释
3. 一个最小例子
4. 最容易混淆的点
5. 常见题型怎么识别
6. 这部分和后续哪里相连

## 4. 直播答疑材料怎么放

- 直播计划：`content/06_live/`
- 直播高频问题：`content/07_faq/`
- 直播回顾页：后续可以同步做成 `docs/live.html` 的更新区块

## 5. 为什么 source 和 docs 分开

- 以后换静态站点工具时，不用重写全部源稿
- 可以先写，再慢慢决定哪些值得公开
- 整个 `mathKnowledge/` 目录可以独立复制走，不依赖当前研究站结构
