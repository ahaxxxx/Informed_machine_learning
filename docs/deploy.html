Deploy

把这套骨架发布到 GitHub Pages

当前推荐方案是 `GitHub Actions` 发布：源文件继续放在仓库里，Actions 负责生成笔记页面并把 `docs/` 目录作为 Pages artifact 部署出去。

基础发布步骤

  1. 把当前仓库推到 GitHub。
  2. 进入仓库 `Settings -> Pages`。
  3. 在 `Build and deployment` 里把 `Source` 切到 `GitHub Actions`。
  4. 确认仓库里已经存在 `.github/workflows/deploy-pages.yml`。
  5. 推送到 `main` 后，等待 `Actions -> Deploy GitHub Pages` 首次跑完。
  6. 部署完成后，再访问 `research.bozhanli.com/notes/...` 之类的新页面路径。

这条流水线具体做了什么

  1. Checkout 当前仓库。
  2. 安装 Python 和 Markdown 转换依赖。
  3. 运行 `python tools/generate_notes_site.py` 生成站内笔记 HTML。
  4. 把 `docs/` 目录上传为 Pages artifact。
  5. 由 `actions/deploy-pages` 正式部署到 GitHub Pages。

自定义域名

  1. 在 GitHub Pages 设置里填入 `research.bozhanli.com`。
  2. 在你的 DNS 提供商处添加一条 `CNAME`：`research -> ahaxxxx.github.io`。
  3. 等待 DNS 生效后，打开 `Enforce HTTPS`。

GitHub 官方文档支持自定义域名和子域名；`research.bozhanli.com` 这种子域名就是典型用法。

要提前知道的限制

如果你之前用的是仓库根目录或 `main /docs` 的 branch 发布，请先把 Pages source 改成 `GitHub Actions`。 不改这个设置，线上就还会继续显示旧页面，而不是新的 `notes/` 页面。
