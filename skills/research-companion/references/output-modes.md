# Output Modes

Choose one output mode before writing. The wrong structure wastes time and hides the real contribution.

## Mode A: New Paper Integration Note

Use when the user wants to bring one paper into an existing system.

Required outputs:

1. one-sentence positioning;
2. eight-question integration note;
3. closest existing paper and key difference;
4. main assumption and failure mode;
5. one concrete research interface left open.

If a local intake template exists, use it directly.

## Mode B: Comparative Synthesis

Use when the user wants several papers compared under one framework.

Recommended structure:

1. common question they are all trying to answer;
2. comparison table with stable axes;
3. one paragraph per axis, not one paragraph per paper;
4. a synthesis paragraph stating what the field still cannot do well.

Stable axes to prefer:

- knowledge object;
- formalization;
- injection interface;
- optimization target;
- assumptions and cost;
- failure mode;
- value for reproduction;
- value for future research.

## Mode C: Literature Review Draft

Use when the user wants a readable review section or survey-style note.

Preferred structure:

1. problem framing;
2. why the split used in the review is the right split;
3. one section per method family or interface family;
4. cross-family comparison;
5. unresolved tensions;
6. targeted research gaps.

Avoid:

- chronological "paper A, paper B, paper C";
- stacking abstracts with weak synthesis;
- calling everything "important" without discrimination.

For beginner-facing review writing:

- define notation locally;
- keep formulas minimal but complete;
- add one intuition sentence after each nontrivial formula block.

## Mode D: Research Gap Memo

Use when the user asks what to work on next.

Required outputs:

1. current local map in 3 to 6 sentences;
2. what the existing papers jointly solve well;
3. what remains structurally unresolved;
4. 3 to 5 candidate research directions;
5. for each direction: entry cost, toy feasibility, novelty risk, and likely evaluation path.

The goal is not "more future work bullets". The goal is decision support.

## Mode E: Study or Review Navigation Note

Use when the user wants a practical document for future querying, review, or revision.

Required outputs:

1. what to read when a certain question appears;
2. what to revisit for formulas;
3. what to revisit for critique;
4. what to revisit for reproduction;
5. what to revisit when designing a new project.

This mode is the right one for meta-notes and handbook-style documents.
